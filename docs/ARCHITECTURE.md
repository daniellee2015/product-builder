## 核心概念

### 一句话总结

**Product Builder = 项目经理（PM/Traffic）+ 开发团队（CCB/CCA）**

### 两个独立的工作流

想象你是一个项目经理，手下有一个开发团队。

#### 工作流1：项目经理的工作（调度工作流）

**你要做什么：**
1. 听用户说需求
2. 把大需求拆成小任务（jobs）
3. 决定先做哪个，后做哪个（优先级）
4. 盯着进度，看有没有卡住的地方（瓶颈识别）
5. 跟用户汇报进展（状态报告）

**谁来做：**
- **选项A**：用OpenClaw这个工具（外部Agent）
- **选项B**：Product Builder自己做（CodeAct模式）

**重点：只能选一个！**

#### 工作流2：开发团队的工作（开发工作流）

**开发团队要做什么：**
1. **Phase 0 - Requirements**：收集需求细节
2. **Phase 1 - Planning**：做技术方案
3. **Phase 2 - Execution**：写代码实现
4. **Phase 3 - Review**：测试和审查
5. **Phase 4 - Archive**：整理文档归档
